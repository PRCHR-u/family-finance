# 🔒 Pre-Launch Security Checklist

## ✅ Критические настройки (обязательно перед запуском)

### 1. Секретные ключи и пароли
- [ ] **SECRET_KEY**: Замените дефолтное значение на криптографически стойкий ключ
  ```bash
  # Генерация надежного ключа
  python -c "import secrets; print(secrets.token_urlsafe(32))"
  ```
- [ ] **ADMIN_PASSWORD**: Смените пароль администратора по умолчанию (`ChangeMe123!`)
- [ ] Убедитесь, что `.env` файл добавлен в `.gitignore` и не закоммичен

### 2. Переменные окружения для Production
Создайте файл `.env` со следующими значениями:
```env
# Обязательные
SECRET_KEY=<ваш-уникальный-ключ-минимум-32-символа>
ADMIN_EMAIL=admin@yourdomain.com
ADMIN_PASSWORD=<надежный-пароль-минимум-12-символов>

# Опциональные
ACCESS_TOKEN_EXPIRE_MINUTES=30
DATABASE_URL=sqlite:///./data/family_finance.db
HOST=0.0.0.0
PORT=8000
```

### 3. База данных
- [ ] Инициализируйте базу данных перед первым запуском:
  ```bash
  python -c "from app.database import init_db; init_db()"
  ```
- [ ] Проверьте права доступа к файлу БД (только чтение/запись для владельца)
- [ ] Настройте регулярное резервное копирование БД

### 4. Сетевая безопасность
- [ ] Ограничьте доступ к порту 8000 только с доверенных IP (если возможно)
- [ ] Используйте reverse proxy (nginx/Apache) в production
- [ ] Настройте SSL/TLS сертификат (Let's Encrypt)
- [ ] Включите HTTPS редирект

### 5. CORS настройки
Проверьте `app/main.py` и обновите `allow_origins`:
```python
allow_origins=[
    "https://yourdomain.com",
    "https://www.yourdomain.com",
]
```
- [ ] Удалите `http://localhost:5173` из списка в production
- [ ] Не используйте `allow_origins=["*"]` с credentials

## 🔐 Рекомендации по безопасности

### Аутентификация и авторизация
- [ ] Минимальная длина пароля: 12 символов
- [ ] Требуйте сложные пароли (заглавные, строчные, цифры, спецсимволы)
- [ ] Реализуйте rate limiting для `/auth/login` (защита от brute-force)
- [ ] Добавьте 2FA для администраторов (опционально)
- [ ] Срок действия JWT токена: не более 30 минут

### Логирование и мониторинг
- [ ] Настройте логирование всех критических действий:
  - Вход/выход пользователей
  - Изменение паролей
  - Создание/удаление записей
  - Ошибки аутентификации
- [ ] Подключите мониторинг (Prometheus/Grafana или аналоги)
- [ ] Настройте алерты при подозрительной активности

### Обновления и зависимости
- [ ] Регулярно обновляйте зависимости:
  ```bash
  pip list --outdated
  pip install --upgrade -r requirements.txt
  ```
- [ ] Проверяйте уязвимости через `pip-audit` или `safety`:
  ```bash
  pip install pip-audit
  pip-audit -r requirements.txt
  ```
- [ ] Подпишитесь на security advisory используемых библиотек

### Docker безопасность (если используется)
- [ ] Не запускайте контейнер от root (добавьте пользователя в Dockerfile)
- [ ] Используйте multi-stage build для уменьшения размера образа
- [ ] Сканируйте образ на уязвимости:
  ```bash
  docker scan family-finance
  ```
- [ ] Ограничьте ресурсы контейнера (CPU, memory)

### Резервное копирование
- [ ] Настройте автоматическое бэкапирование:
  - База данных: ежедневно
  - Логи: еженедельно
  - Конфигурация: при изменениях
- [ ] Храните бэкапы отдельно от основного сервера
- [ ] Тестируйте восстановление из бэкапа раз в квартал

## 🚀 Deployment Checklist

### Перед деплоем
- [ ] Все тесты проходят успешно
- [ ] Код проверен через линтеры (flake8, black, mypy)
- [ ] Удалены debug prints и временный код
- [ ] Отключен debug режим FastAPI
- [ ] Проверены все endpoint'ы через Swagger UI

### После деплоя
- [ ] Проверьте health endpoint: `GET /health`
- [ ] Протестируйте регистрацию нового пользователя
- [ ] Протестируйте вход под администратором
- [ ] Создайте тестовую запись и проверьте модерацию
- [ ] Проверьте логи на наличие ошибок
- [ ] Замерьте время ответа API (< 500ms для простых запросов)

## 📊 Мониторинг после запуска

### Ежедневные проверки
- [ ] Статус сервиса (uptime > 99%)
- [ ] Количество ошибок в логах
- [ ] Потребление памяти и CPU
- [ ] Размер базы данных

### Еженедельные проверки
- [ ] Анализ аудита безопасности
- [ ] Проверка неудачных попыток входа
- [ ] Обновление зависимостей (при наличии патчей безопасности)
- [ ] Ревью логов на аномалии
