# Family Finance

Многопользовательское приложение для учета и планирования финансов с подтверждением изменений администратором.

## Реализовано

- JWT-аутентификация (регистрация, вход, bearer token).
- Роли:
  - `user` — добавляет/редактирует записи;
  - `admin` — подтверждает/отклоняет записи, импортирует Excel, управляет пользователями.
- Финансовые поля:
  - `debt_total` — общий долг;
  - `urgent_creditcard_repayment` — срочное погашение кредиток;
  - `income` — доход;
  - `mandatory_expense` — обязательные расходы;
  - `planned_expense` — планируемые расходы.
- Workflow согласования:
  - изменения пользователей создаются/переводятся в `pending`;
  - админ подтверждает (`approve`) или отклоняет (`reject`).
- Аналитика:
  - изменение долга за период (на основе истории долгов и погашений);
  - стартовый/конечный долг;
  - сумма доходов, обязательных и срочных погашений.
- Детализированный учет долгов:
  - отдельные долги (`creditor_name`, сумма, дата, ставка, план погашения);
  - частичные погашения (`DebtRepayment`) с модерацией админом;
  - автоматический пересчет остатка по долгу и статуса (`active`/`closed`).
- Простой веб-интерфейс на `http://127.0.0.1:8000/`.
  - Включает админ-панель: создание пользователей, активация/деактивация.
  - Добавлены редактирование/удаление пользователей и аудит-лог действий.
  - Аудит поддерживает фильтры, пагинацию и экспорт в CSV.

## Импорт из Excel

Добавлен endpoint `POST /imports/excel` (только админ), который читает `ДОЛГИ.xlsx` и переносит:

- дату периода;
- общий долг;
- обязательные расходы;
- срочное погашение;
- планируемый расход;
- доход (из блока доходов в таблице).

## Запуск

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

- Swagger: `http://127.0.0.1:8000/docs`
- Web UI: `http://127.0.0.1:8000/`

## Учетная запись администратора по умолчанию

Создается автоматически при первом запуске:

- `username`: `admin`
- `password`: `admin123456`

После первого входа рекомендуется сменить пароль (в текущем MVP можно сделать через создание нового admin пользователя и удаление старого в БД).
