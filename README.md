# Family Finance

MVP backend для многопользовательского учета и планирования финансов.

## Что реализовано

- Роли пользователей: `admin` и `user`.
- Записи с ключевыми полями из Excel:
  - общий долг (`debt_total`);
  - срочное погашение по кредиткам в льготный период (`urgent_creditcard_repayment`);
  - доход (`income`);
  - обязательные расходы (`mandatory_expense`);
  - планируемые расходы (`planned_expense`).
- Модерация администратором:
  - записи и изменения пользователей создаются в статусе `pending`;
  - администратор подтверждает (`approve`) или отклоняет (`reject`).
- Аналитика по периоду:
  - общий долг на начало и конец периода;
  - изменение долга за период;
  - суммарный доход, обязательные расходы и срочные погашения.

## Быстрый запуск

1. Создать и активировать виртуальное окружение.
2. Установить зависимости:
   - `pip install -r requirements.txt`
3. Запустить API:
   - `uvicorn app.main:app --reload`
4. Открыть Swagger:
   - `http://127.0.0.1:8000/docs`

## Авторизация (MVP)

Используется заголовок `X-User-Id` для определения текущего пользователя.

- При первом запуске автоматически создается администратор:
  - `username`: `admin`
  - `id`: `1` (если база новая)

Пример заголовка для запросов:

`X-User-Id: 1`

## Основные endpoints

- `POST /users` — создать пользователя (только админ).
- `GET /users/me` — получить текущего пользователя.
- `POST /records` — добавить финансовую запись.
- `PUT /records/{record_id}` — изменить запись.
- `GET /records` — список записей (пользователь видит только свои).
- `POST /records/{record_id}/approve` — подтверждение записи админом.
- `POST /records/{record_id}/reject` — отклонение записи админом.
- `GET /analytics/debt-summary?period_from=YYYY-MM-DD&period_to=YYYY-MM-DD` — сводка долга и ключевых сумм за период.
