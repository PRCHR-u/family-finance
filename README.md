# Family Finance

Многопользовательское приложение для учета и планирования семейных финансов с подтверждением изменений администратором.

## Возможности

### 🔐 Аутентификация и авторизация
- JWT-аутентификация (регистрация, вход, bearer token)
- Смена пароля пользователем
- Ролевая модель:
  - `user` — добавляет/редактирует записи;
  - `admin` — подтверждает/отклоняет записи, импортирует Excel, управляет пользователями.

### 👥 Управление пользователями (только admin)
- Создание новых пользователей с указанием роли
- Редактирование пользователей (логин, роль, пароль, статус активности)
- Активация/деактивация пользователей
- Удаление пользователей (без финансовых записей)
- Просмотр списка всех пользователей

### 💰 Финансовые записи
- **Доходы** (`income`) — планируемые и фактические
- **Обязательные расходы** (`mandatory_expense`) — ежемесячные обязательные платежи
- **Планируемые расходы** (`planned_expense`) — необязательные запланированные траты
- **Срочное погашение кредиток** (`urgent_creditcard_repayment`)
- **Общий долг** (`debt_total`) — агрегированная информация

### 📊 Детализированный учет долгов
- Отдельные долги с полями:
  - `creditor_name` — название кредитора
  - `principal_amount` — сумма долга
  - `start_date` — дата начала
  - `interest_rate` — процентная ставка
  - `repayment_plan` — план погашения
- Частичные погашения (`DebtRepayment`) с модерацией админом
- Автоматический пересчет остатка по долгу и статуса (`active`/`closed`)
- История погашений с фильтрацией

### 💳 Учет кредитных карт
- Карточка с датой старта льготного периода и его длительностью
- Автоматический расчет:
  - Даты окончания льготного периода
  - Оставшихся дней до конца льготного периода
- Endpoint «Срочно погасить» — топ-3 карты, ближайшие к дедлайну
- Статусы карт: `active`, `closed`

### ✅ Workflow согласования
- Изменения пользователей создаются/переводятся в статус `pending`
- Админ подтверждает (`approve`) или отклоняет (`reject`) записи
- Поддерживаемые сущности:
  - Финансовые записи
  - Долги и погашения
  - Кредитные карты
  - Доходы и расходы

### 📈 Аналитика и отчеты
- **Анализ изменения долга за период**:
  - Стартовый и конечный долг
  - Сумма новых долгов и погашений
  - Чистое изменение долга
  - Детализация по новым долгам и платежам
- **Недельный бюджет**:
  - Общий и средний дневной бюджет
  - Обязательные и планируемые расходы
  - Доступный доход и баланс
  - Рекомендации по бюджету
- **Ежедневный бюджет**:
  - Обязательные расходы на день
  - Базовый и дискреционный бюджет
  - Персональные рекомендации
- **Сводка бюджета за период**:
  - Анализ долгов
  - Обзор бюджета (средние значения)
  - Оценка финансового здоровья:
    - Тренд долга (растет/снижается/стабилен)
    - Баланс бюджета (профицит/дефицит/сбалансирован)
- **Ежемесячные обязательные расходы** — сводка по категориям

### 📥 Импорт из Excel
Endpoint `POST /imports/excel` (только админ) читает файл `ДОЛГИ.xlsx` и импортирует:
- Дату периода
- Общий долг
- Обязательные расходы
- Срочное погашение
- Планируемый расход
- Доход (из блока доходов в таблице)

### 📝 Аудит-лог
- Логирование всех значимых действий:
  - Регистрация пользователей
  - Создание/изменение/удаление пользователей
  - Активация/деактивация
  - Смена пароля
  - Операции с долгами, погашениями, кредитными картами
  - Подтверждение/отклонение записей
- Фильтрация по:
  - Типу действия
  - Пользователю-исполнителю
  - Целевому пользователю
  - Периоду
- Пагинация результатов
- Экспорт в CSV

### 🌐 Веб-интерфейс
Простой веб-интерфейс на `http://127.0.0.1:8000/`:
- Админ-панель: создание пользователей, активация/деактивация
- Редактирование и удаление пользователей
- Аудит-лог с фильтрами, пагинацией и экспортом в CSV

## Запуск

### Вариант 1: Автоматический деплой (рекомендуется)

```bash
./deploy.sh
```

Скрипт автоматически настроит виртуальное окружение, установит зависимости, сгенерирует SECRET_KEY и предложит выбрать режим запуска.

### Вариант 2: Ручная установка

```bash
# Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
pip install gunicorn

# Настройка переменных окружения
cp .env.example .env
# Отредактируйте .env, измените пароли и сгенерируйте SECRET_KEY

# Инициализация БД
python3 -c "from app.database import init_db; init_db()"

# Запуск сервера
gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
```

### Вариант 3: Docker

```bash
docker-compose up -d
docker-compose logs -f
```

## Учетная запись администратора по умолчанию

Создается автоматически при первом запуске:

- `email`: `admin@example.com` (настраивается через `ADMIN_EMAIL`)
- `password`: `ChangeMe123!` (настраивается через `ADMIN_PASSWORD`)

**Важно:** После первого входа сразу смените пароль через веб-интерфейс или API endpoint `/users/change-password`.

Для изменения учетных данных администратора используйте переменные окружения:
- `ADMIN_EMAIL` — email администратора
- `ADMIN_PASSWORD` — пароль администратора
